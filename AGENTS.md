# Agent Rules

## 最重要
- いきなり実装しない。まず対象ファイル、変更方針、影響範囲を短く説明する。
- 推測で仕様を作らない。不明点は「不明」と書き、既存コードから確認する。
- 一度に大改修しない。原則、1タスク1目的。
- 完了宣言の前に必ず検証結果を出す。

## 技術スタック
- HTML / JavaScript ES Modules / Tailwind CDN / Chart.js CDN の範囲で実装する。
- ビルドツール、フレームワーク、不要な依存追加は禁止。
- コピー＆ペーストまたは静的ホスティングで動く構成を維持する。

## 実装前チェック
- 変更対象ファイル
- 変更理由
- 壊れそうな箇所
- テスト方法

## 完了条件
- 変更内容の要約
- 実行した確認コマンド
- ブラウザ上で確認すべき操作
- 残った懸念点
