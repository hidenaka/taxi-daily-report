# ホーム画面「目標と実績」見やすさ改善 — 設計

作成: 2026-05-18

## 背景・課題
ホーム画面の実績カード（summaryCard）と目標カード（targetCard）に数値が約35個並び、
実績・予想・目標・歩率が同じ見た目の行で混在。一目で要点が掴めない。

## 方針
「見出し＋折りたたみ」。最優先情報（今の実績）を大きく常時表示し、細目はタップで開く折りたたみに収める。
計算ロジック・数値は一切変えない。**HTMLの並べ替えと折りたたみ化のみ。**

## 確定デザイン（案B = 2枚カード維持）

### 実績カード（summaryCard）
- 常時表示: 月切替ナビ／`現時点 手取り`（特大ヒーロー）／出番数 `7 / 12`／`総支給`／`月度予想 手取り`（中サイズ・当月のみ）
- 折りたたみ ▸:
  - `給与の内訳` — renderPayrollBreakdown の内容
  - `営収・1出番平均` — 月度合計営収(税込/税抜)・1出番平均(税込/税抜・歩率判定)・月度予想総支給
  - `歩率と上げ／下げの境界` — 歩率(暫定/確定)＋▲/▲▲/▼ブロック。見出しに現在歩率バッジ
- 過去月: ヒーロー=確定手取り、予想なし。未来月: 予定出番のみ。

### 目標カード（targetCard）
- 常時表示: `手取り目標`額／`達成見込`・`未達見込`／進捗バー／あと差額
- 折りたたみ ▸:
  - `あと必要な売上` — 残り乗務・残り必要総売上(税込/税抜)・1乗務あたり必要売上(税込/税抜)
  - `目標の細目` — renderSubTargetCards（総支給/11出番/12出番以降/合計）

## 動作
- 折りたたみは `<details>` ネイティブ要素。初期状態すべて閉。
- 開閉状態は localStorage（キー `homefold:<id>`）に記憶。
- 中身が空のセクションは出さない。

## 実装範囲
- `index.html` のみ（render() の summaryHtml / targetCard 生成、renderPayrollBreakdown、CSS、foldSection/wireFolds 追加）
- `sw.js` キャッシュ版数 bump
- テスト: 既存216件（payroll/parser）に影響なし。表示確認は dev で実施。
